## Overview

QWED-AI provides a robust REST API at `https://api.qwed.ai/v1` for verifying LLM outputs across math, logic, finance, and code domains. Use the Verifier class in the Python SDK for streamlined integration, or make direct HTTP calls. All endpoints require authentication via `Authorization: Bearer {YOUR_API_KEY}` header.

<Callout kind="info">
API keys are available in your dashboard at `https://dashboard.qwed.ai`. Rate limits apply: 100 requests per minute.
</Callout>

## Verifier Class (Python SDK)

Install via PyPI: `pip install qwed`.

```python

verifier = Verifier(api_key="{YOUR_API_KEY}")
```

## Verification Endpoints

### Math Verification

Verify mathematical statements from LLM outputs.

<Request tabs="Python,cURL" show-lines="true">
  ```python
  result = verifier.verify_math(
      statement="2 + 2 = 4",
      context="Basic arithmetic check"
  )
  print(result.is_valid)  # True
  ```
  ```bash
  curl -X POST https://api.qwed.ai/v1/verify/math \
    -H "Authorization: Bearer {YOUR_API_KEY}" \
    -H "Content-Type: application/json" \
    -d '{
      "statement": "2 + 2 = 4",
      "context": "Basic arithmetic check"
    }'
  ```
</Request>

<Response tabs="200,400">
  ```json
  {
    "is_valid": true,
    "confidence": 1.0,
    "explanation": "The equation 2 + 2 equals 4 is correct.",
    "proof_steps": ["Addition: 2 + 2", "Result: 4"]
  }
  ```

  ```json
  {
    "is_valid": false,
    "error": "Invalid statement format"
  }
  ```
</Response>

#### Request Parameters

<ParamField body="statement" param-type="string" required="true">
Mathematical expression to verify, e.g., `{2 + 2 = 4}`.
</ParamField>

<ParamField body="context" param-type="string" required="false">
Additional context for the proof.
</ParamField>

#### Response Fields

<ResponseField name="is_valid" field-type="boolean" required="true">
Whether the statement is verified.
</ResponseField>

<ResponseField name="confidence" field-type="number" required="true">
Verification confidence score (0.0 to 1.0).
</ResponseField>

<ResponseField name="explanation" field-type="string">
Human-readable proof explanation.
</ResponseField>

## Error Handling

All API responses use standard HTTP status codes. Errors return JSON with `error` field.

<Callout kind="alert">
Handle `401 Unauthorized` by refreshing your API key. Retry `429 Rate Limit` after `Retry-After` seconds.
</Callout>

Common errors:

| Code | Message              | Action                  |
|------|----------------------|-------------------------|
| 400  | Invalid statement   | Check payload format   |
| 401  | Unauthorized        | Verify `Authorization` header |
| 429  | Rate limit exceeded | Wait and retry         |
| 500  | Internal server error | Contact support       |

```python
try:
    result = verifier.verify_math("invalid")
except APIError as e:
    print(f"Error: {e.message}")  # e.g., "Invalid statement format"
```
